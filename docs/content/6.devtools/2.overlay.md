## Installation

Add `qora_devtools_overlay` to `dev_dependencies`. This keeps the package out of production bundles.

```yaml [pubspec.yaml]
dev_dependencies:
  qora_devtools_overlay: ^1.0.0
```

---

## Setup

Create an `OverlayTracker` and pass it to both `QoraClient` and `QoraInspector`. Pass the client reference to `QoraInspector` to enable the inspector action buttons.

```dart [main.dart]
import 'package:flutter/foundation.dart';
import 'package:qora/qora.dart';
import 'package:qora_devtools_overlay/qora_devtools_overlay.dart';

void main() {
  final tracker = OverlayTracker();

  final client = QoraClient(
    // null in release mode: QoraClient uses NoOpTracker, no events emitted.
    tracker: kDebugMode ? tracker : null,
  );

  runApp(
    QoraInspector(
      tracker: tracker,
      // Required for action buttons (Refetch, Invalidate, Remove, etc.)
      // to operate on the live cache. Omitting this disables the ACTIONS section.
      client: kDebugMode ? client : null,
      child: MyApp(client: client),
    ),
  );
}
```

In debug mode, a **Q** button is rendered in the bottom-right corner of the screen. Tapping it opens the panel.

::callout{type="warning"}
The `client` parameter is optional. `QoraInspector` operates as a read-only observer when `client` is not provided. The **ACTIONS** section is not rendered in that case.
::

::callout{type="info"}
`QoraInspector` renders nothing in release builds. The guard is applied at widget level: the overlay subtree is not constructed when `kDebugMode` is false.
::

### With the IDE extension

When both the overlay and the IDE extension are needed simultaneously, use `QoraDevtools.setup` with `additionalTrackers`. Both trackers receive identical events via a shared `MultiTracker`.

```dart [main.dart]
import 'package:flutter/foundation.dart';
import 'package:qora_devtools_extension/qora_devtools_extension.dart';
import 'package:qora_devtools_overlay/qora_devtools_overlay.dart';

void main() {
  final overlay = OverlayTracker();
  final client  = QoraClient();

  if (kDebugMode) {
    QoraDevtools.setup(client, additionalTrackers: [overlay]);
  }

  runApp(
    QoraInspector(
      tracker: overlay,
      client: kDebugMode ? client : null,
      child: MyApp(client: client),
    ),
  );
}
```

## Responsive behavior

The overlay layout adapts to the available screen width.

- **600 px and above** (tablet, desktop): three-column layout with all panels visible simultaneously.
- **Below 600 px** (phone): single-column navigation with forward navigation into inspector and timeline views.

## Production safety

Three independent mechanisms ensure the overlay is absent from production:

### 1. `dev_dependencies`

The package is not included in the release bundle because it is declared under `dev_dependencies`.

### 2. Widget guard

```dart
@override
Widget build(BuildContext context) {
  if (!kDebugMode) return child; // nothing is built in release
  return _buildOverlay();
}
```

### 3. NoOpTracker in release

`QoraClient` is constructed with `tracker: kDebugMode ? tracker : null`. In release mode, `null` activates `NoOpTracker`. Its methods are no-ops and no tracking events are emitted.

Result: no runtime impact from the overlay tracker path in production.
