## Installation

Add `qora_devtools_overlay` to `dev_dependencies`, not `dependencies`. This keeps the package out of production bundles.

```yaml [pubspec.yaml]
dev_dependencies:
  qora_devtools_overlay: ^0.1.0
```

---

## Setup

Create an `OverlayTracker`, pass it to `QoraClient`, `QoraInspector`, and тАФ to enable the inspector action buttons тАФ pass the client reference to `QoraInspector` as well.

```dart [main.dart]
import 'package:flutter/foundation.dart';
import 'package:qora/qora.dart';
import 'package:qora_devtools_overlay/qora_devtools_overlay.dart';

void main() {
  final tracker = OverlayTracker();

  final client = QoraClient(
    // null in release тЖТ QoraClient uses NoOpTracker, no events emitted.
    tracker: kDebugMode ? tracker : null,
  );

  runApp(
    QoraInspector(
      tracker: tracker,
      // Required for action buttons (Refetch, Invalidate, Remove, etc.)
      // to operate on the live cache. Omitting this hides the ACTIONS section.
      client: kDebugMode ? client : null,
      child: MyApp(client: client),
    ),
  );
}
```

In debug mode, a floating **Q** button is rendered in the bottom-right corner of the app.

::callout{type="warning"}
The `client` parameter is optional by design. `QoraInspector` functions as a read-only observer without it. When omitted, the **ACTIONS** section in the query inspector is not rendered.
::

::callout{type="info"}
`QoraInspector` renders nothing in release builds. The guard is applied at widget level, so the overlay subtree is not constructed outside `kDebugMode`.
::

## Responsive behavior

The overlay adapts to the available screen size.

- **тЙе 600 px** (tablet, desktop): Three-column layout with all panels visible.
- **< 600 px** (phone): Single-column navigation with forward navigation into inspector and timeline views.

## Production safety

Three independent layers ensure the overlay never reaches production:

### 1. `dev_dependencies`

The package is physically absent from your release bundle because it lives in `dev_dependencies`.

### 2. Widget guard

```dart
@override
Widget build(BuildContext context) {
  if (!kDebugMode) return child; // nothing is built in release
  return _buildOverlay();
}
```

### 3. NoOpTracker in release

`QoraClient` is constructed with `tracker: kDebugMode ? tracker : null`. In release mode, `null` activates `NoOpTracker`. Its methods are no-ops and no tooling events are emitted.

Result: no production runtime impact from the overlay tracker path.
