Qora provides two complementary developer tooling surfaces. Both use the same event protocol and require no production configuration changes.

::card-group
  ::card
  ---
  title: In-App Overlay
  icon: i-heroicons-squares-2x2
  to: /devtools/overlay
  ---
  A draggable panel inside the running app. This surface is removed from release builds.
  ::

  ::card
  ---
  title: IDE Extension
  icon: i-heroicons-command-line
  to: /devtools/ide-extension
  ---
  A native tab inside Flutter DevTools for VS Code and IntelliJ. Inspect the full cache and trigger commands directly from the IDE.
  ::
::

## Comparison

|                              | In-App Overlay                         | IDE Extension                                    |
| ---------------------------- | -------------------------------------- | ------------------------------------------------ |
| **Setup**                    | Wrap your app with `QoraInspector`     | Add `qora_devtools_extension` to dependencies    |
| **Where it runs**            | Inside your app                        | Inside Flutter DevTools (VS Code / IntelliJ)     |
| **Transport**                | Direct, same process                    | Dart VM Service (WebSocket)                      |
| **Mutations timeline**       | ✅                                     | ✅                                               |
| **Cache inspector**          | ✅                                     | ✅ live + snapshot                               |
| **Optimistic updates**       | ✅                                     | ✅                                               |
| **Data Diff**                | ✅                                     | ✅                                               |
| **Refetch / Invalidate**     | ✅                                     | ✅                                               |
| **Network Activity Monitor** | Not available                           | ✅                                               |
| **Performance Metrics**      | Not available                           | ✅                                               |
| **Dependency Graph**         | Not available                           | ✅                                               |
| **Widget Tree**              | Not available                           | Planned                                          |
| **Production overhead**      | None (`kDebugMode` guard)              | None (stripped by tree-shaker)                   |
| **Works on physical device** | ✅                                     | ✅ (via USB)                                     |

## How it works

Both surfaces consume the event stream produced by `QoraTracker`, an interface implemented by `QoraClient`. In production, omit the tracker. `QoraClient` then defaults to `NoOpTracker`, with zero runtime tracking overhead.

```
QoraClient
    │
    ├── NoOpTracker (production)     → silent, inlined away by the compiler
    ├── VmTracker  (IDE extension)   → emits events over Dart VM Service
    └── OverlayTracker (overlay)     → emits events in memory
```

Pass the tracker explicitly when constructing `QoraClient`. Omit it in production to use `NoOpTracker` automatically.
