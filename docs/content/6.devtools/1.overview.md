Qora provides two developer tooling surfaces. Both consume the same event protocol. Neither requires configuration changes in production builds.

## Comparison

|                              | In-App Overlay                         | IDE Extension                                    |
| ---------------------------- | -------------------------------------- | ------------------------------------------------ |
| **Setup**                    | Wrap the app with `QoraInspector`      | Call `QoraDevtools.setup(client)`                |
| **Where it runs**            | Inside the app process                 | Inside Flutter DevTools (VS Code / IntelliJ)     |
| **Transport**                | Direct, same process                   | Dart VM Service (WebSocket)                      |
| **Mutations timeline**       | ✅                                     | ✅                                               |
| **Cache inspector**          | ✅                                     | ✅ live + snapshot                               |
| **Optimistic updates**       | ✅                                     | ✅                                               |
| **Data Diff**                | ✅                                     | ✅                                               |
| **Refetch / Invalidate**     | ✅                                     | ✅                                               |
| **Network Activity Monitor** | Not available                          | ✅                                               |
| **Performance Metrics**      | Not available                          | ✅                                               |
| **Dependency Graph**         | Not available                          | ✅                                               |
| **Widget Tree**              | Not available                          | Planned                                          |
| **Production overhead**      | None (`kDebugMode` guard)              | None (stripped by tree shaker)                   |
| **Works on physical device** | ✅                                     | ✅ (via USB)                                     |

## How it works

Both surfaces consume the event stream produced by `QoraTracker`, the observability interface of `QoraClient`. In production, no tracker is configured. `QoraClient` defaults to `NoOpTracker`, which has zero runtime overhead.

```
QoraClient
    │
    ├── NoOpTracker    (production)     silent, no events emitted
    ├── VmTracker      (IDE extension)  emits events over Dart VM Service
    └── OverlayTracker (overlay)        emits events in memory
```

For the IDE extension, configure the tracker via `QoraDevtools.setup`. For the overlay alone, pass the tracker directly to the `QoraClient` constructor. In production, pass `null` or omit the `tracker` parameter to activate `NoOpTracker`.
