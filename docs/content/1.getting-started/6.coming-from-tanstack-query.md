Qora shares the same conceptual model as TanStack Query: query keys, stale-while-revalidate, background refetch, mutation lifecycle hooks, and offline pause-and-replay. Most concepts translate directly. This page maps what you already know to the Qora API.

---

## Concept Map

| TanStack Query concept | Qora equivalent |
| ---------------------- | --------------- |
| `QueryClient` | `QoraClient` |
| `QueryClientProvider` | `QoraScope` |
| `useQuery` | `QoraBuilder<T>` (widget) or `useQuery<T>` (hook) |
| `useMutation` | `QoraMutationBuilder` (widget) or `useMutation` (hook) |
| `useInfiniteQuery` | `InfiniteQueryBuilder` (widget) or `useInfiniteQuery` (hook) |
| `useQueryClient` | `context.qora` / `useQueryClient()` (hook) |
| `useIsFetching` | `useIsFetching()` (hook) or `client.fetchingCountStream` |
| `useIsMutating` | `useIsMutating()` (hook) or `client.activeMutations` |
| `QueryObserver` | `client.watchQuery<T>` / `InfiniteQueryObserver` |
| `persistQueryClient` | `PersistQoraClient` |
| `DevTools` | `qora_devtools_overlay` (in-app) / IDE extension |

## API Reference: Queries

### Options

| TanStack Query option | Qora equivalent | Notes |
| --------------------- | --------------- | ----- |
| `queryKey` | `queryKey` | Same concept. TanStack uses arrays; Qora uses `List<dynamic>`. Structural equality in both. |
| `queryFn` | `fetcher` | Same role. Renamed. |
| `enabled` | `enabled` | Identical. |
| `staleTime` | `staleTime` | Identical. `Duration.zero` = always stale. |
| `gcTime` (`cacheTime` in v4) | `cacheTime` | Qora uses the v4 name. |
| `retry` | `retryCount` | Renamed. Default is `3` in both. |
| `retryDelay` | `retryDelay` / `retryDelayCalculator` | Exponential backoff is the default. `retryDelayCalculator` accepts a function receiving the attempt index. |
| `refetchOnWindowFocus` | `refetchOnWindowFocus` | Identical. Requires `FlutterLifecycleManager`. |
| `refetchOnReconnect` | `refetchOnReconnect` | Identical. Requires `FlutterConnectivityManager`. |
| `refetchOnMount` | `refetchOnMount` | Identical semantics. `null` falls back to `QoraClientConfig.refetchOnMount`. |
| `refetchInterval` | `refetchInterval` | Identical. Active while at least one subscriber is present. |
| `networkMode` | `networkMode` | Same name, same three values: `online`, `always`, `offlineFirst`. |
| `select` | `state.map()` / `state.mapSuccess()` | No option; use the `map` method on the returned `QoraState<T>` instead. |
| `placeholderData: keepPreviousData` | Built into `Loading` / `Failure` | No option needed. `Loading<T>` always carries `previousData: T?`. The behaviour is always active. |
| `initialData` | `client.hydrateQuery<T>(key, data)` | Pre-populates the cache before the first subscription. |
| `initialDataUpdatedAt` | `client.hydrateQuery<T>(key, data, updatedAt: ...)` | Pass `updatedAt` to control stale evaluation. |
| `structuralSharing` | Not available | Dart does not perform structural sharing by default. |
| `notifyOnChangeProps` | Not available | All state transitions emit on the stream. |

### Return values

| TanStack Query field | Qora equivalent |
| -------------------- | --------------- |
| `status: 'pending'` | `state is Initial` or `state is Loading(previousData: null)` |
| `status: 'error'` | `state is Failure` |
| `status: 'success'` | `state is Success` |
| `fetchStatus: 'fetching'` | `fetchStatus == FetchStatus.fetching` |
| `fetchStatus: 'paused'` | `fetchStatus == FetchStatus.paused` |
| `fetchStatus: 'idle'` | `fetchStatus == FetchStatus.idle` |
| `isLoading` | `state.isLoading` |
| `isFetching` | `fetchStatus == FetchStatus.fetching` |
| `isError` | `state.isError` |
| `isSuccess` | `state.isSuccess` |
| `isPending` | `state is Initial \|\| state is Loading(previousData: null)` |
| `data` | `state.dataOrNull` or `Success(:final data)` via pattern match |
| `error` | `state.errorOrNull` or `Failure(:final error)` via pattern match |
| `dataUpdatedAt` | `state.updatedAt` (extension, non-null only on `Success`) |
| `errorUpdatedAt` | Not available as a separate field |
| `isFetchedAfterMount` | Not available |

## API Reference: Mutations

### Options

| TanStack Query option | Qora equivalent |
| --------------------- | --------------- |
| `mutationFn` | `mutator` | Renamed. |
| `onMutate` | `MutationOptions.onMutate` | Same role. Return value becomes `TContext`. |
| `onSuccess` | `MutationOptions.onSuccess` | Same role. Signature: `(TData, TVariables, TContext?)`. |
| `onError` | `MutationOptions.onError` | Same role. Signature: `(Object, TVariables, TContext?)`. |
| `onSettled` | `MutationOptions.onSettled` | Same role. |
| `retry` | `MutationOptions.retryCount` | Renamed. Default is `0` (no retry). |
| `retryDelay` | `MutationOptions.retryDelay` | Exponential backoff. |
| `networkMode` | Not per-mutation in Qora | Offline queueing is controlled by `MutationOptions.offlineQueue: true`. |
| `variables` | `TVariables` type parameter | Same concept. |
| `context` (rollback snapshot) | `TContext` type parameter | Same concept. Returned by `onMutate`, forwarded to `onError`/`onSettled`. |

### Return values / state

| TanStack Query field | Qora equivalent |
| -------------------- | --------------- |
| `status: 'idle'` | `state is MutationIdle` |
| `status: 'pending'` | `state is MutationPending` |
| `status: 'error'` | `state is MutationFailure` |
| `status: 'success'` | `state is MutationSuccess` |
| `isIdle` | `state.isIdle` |
| `isPending` | `state.isPending` |
| `isError` | `state.isError` |
| `isSuccess` | `state.isSuccess` |
| `data` | `state.dataOrNull` |
| `error` | `state.errorOrNull` |
| `variables` | `state.variablesOrNull` |
| `mutate(variables)` | `mutate(variables)` — fire-and-forget, errors captured in state |
| `mutateAsync(variables)` | `mutateAsync(variables)` — awaitable, errors propagated |
| `reset()` | `reset()` — transitions back to `MutationIdle` |

## API Reference: `QueryClient` → `QoraClient`

| TanStack Query method | Qora equivalent |
| --------------------- | --------------- |
| `invalidateQueries({ queryKey })` | `client.invalidate(key)` |
| `invalidateQueries(filter)` | `client.invalidateWhere(predicate)` or `client.invalidateQueries(filter)` |
| `setQueryData(queryKey, updater)` | `client.setQueryData<T>(key, data)` |
| `getQueryData(queryKey)` | `client.getQueryData<T>(key)` — returns `T?` |
| `getQueryState(queryKey)` | `client.getQueryState<T>(key)` — returns `QoraState<T>` |
| `prefetchQuery` | `client.prefetch<T>(key, fetcher, options)` |
| `resetQueries` | `client.removeQuery(key)` or `client.clear()` |
| `cancelQueries` | Not available in Qora v1 |
| `isFetching(queryKey)` | `client.isFetchingCount` (global count) |
| `setDefaultOptions` | `QoraClientConfig.defaultOptions` (set at construction) |
| `getDefaultOptions` | `client.config.defaultOptions` |
| `mount` / `unmount` | Handled by `QoraScope` / widget lifecycle |
| `clear()` | `client.clear()` |

## Key Differences

### Sealed classes replace status strings

TanStack Query returns plain objects with string `status` fields and boolean shortcuts (`isLoading`, `isError`, etc.). Qora returns a sealed class `QoraState<T>`. Dart 3 pattern matching makes exhaustive handling compile-time enforced:

```dart [user_screen.dart]
// TanStack Query (React)
if (isLoading) return <Spinner />;
if (isError) return <Error error={error} />;
return <UserCard user={data} />;

// Qora (Dart 3)
return switch (state) {
  Initial() || Loading(previousData: null) => const CircularProgressIndicator(),
  Failure(:final error, previousData: null) => ErrorScreen(error),
  _ => UserCard(state.dataOrNull!),
};
```

The `_` arm covers `Success`, `Loading(previousData != null)`, and `Failure(previousData != null)` — all cases where data is available.

---

### `previousData` is always present, not opt-in

TanStack Query v4 had `keepPreviousData: true`. TanStack Query v5 replaced it with `placeholderData: keepPreviousData`. In Qora, `Loading<T>` and `Failure<T>` always carry `previousData: T?`. There is no option to disable this. Stale data is always accessible during background refreshes and after soft errors.

```dart [user_screen.dart]
// Pattern: show stale data with a progress indicator during background refresh
case Loading(:final previousData) when previousData != null:
  return Stack(children: [
    UserCard(previousData),
    const LinearProgressIndicator(minHeight: 2),
  ]);
```

---

### `select` is a method, not an option

TanStack Query has a `select` option to transform the data before returning it to the component. Qora exposes `map` and `mapSuccess` methods on `QoraState<T>`:

```dart [main.dart]
// TanStack Query
useQuery({ queryKey, queryFn, select: (data) => data.name });

// Qora
final nameState = userState.map((user) => user.name);
// QoraState<User> → QoraState<String>
// previousData in Loading/Failure is also transformed
```

---

### `gcTime` is called `cacheTime`

TanStack Query v5 renamed `cacheTime` to `gcTime`. Qora uses `cacheTime` (the v4 name). The semantics are identical: time before an inactive entry is garbage-collected after its last subscriber cancels.

---

### Widget API vs hook API

React Query's primary API is hooks (`useQuery`, `useMutation`). Qora provides both a widget API and a hook API:

- `QoraBuilder<T>` and `QoraMutationBuilder` work in any `StatelessWidget` or `StatefulWidget`.
- `useQuery<T>` and `useMutation` require `HookWidget` from `flutter_hooks`.

Both share the same `QoraClient` cache. If a `QoraBuilder` already fetched `['users', id]`, `useQuery` gets it on the first frame from the same entry.

---

### `QoraScope` is a `StatefulWidget`, not a `Provider`

TanStack Query uses `QueryClientProvider` (React context). Qora uses `QoraScope`, a `StatefulWidget` that exposes the client to its subtree via an internal `InheritedWidget`. Access the client from any descendant via `context.qora` or `QoraScope.of(context)`.

---

### Persistence requires explicit serializer registration

TanStack Query's persistence plugins (e.g. `createSyncStoragePersister`) use `JSON.stringify` / `JSON.parse` on the full query cache. Qora requires explicit serializer registration per type because Dart has no runtime reflection:

```dart [main.dart]
// Required before hydrate()
client.registerSerializer<User>(
  QoraSerializer(toJson: (u) => u.toJson(), fromJson: User.fromJson),
  name: 'User',
);
await client.hydrate();
```

The `name` string must be stable across app versions and build modes (including obfuscated builds).

---

### No React Suspense equivalent

TanStack Query supports `suspense: true` to integrate with React Suspense. Flutter has no equivalent mechanism. All async state is handled through `QoraState<T>` pattern matching in the widget tree.

## Infinite Queries

| TanStack Query | Qora |
| -------------- | ---- |
| `useInfiniteQuery` | `InfiniteQueryBuilder` / `useInfiniteQuery` |
| `queryFn({ pageParam })` | `fetcher(TPageParam pageParam)` |
| `initialPageParam` | `InfiniteQueryOptions.initialPageParam` |
| `getNextPageParam(lastPage, pages)` | `InfiniteQueryOptions.getNextPageParam(lastPage, pages)` |
| `getPreviousPageParam` | `InfiniteQueryOptions.getPreviousPageParam` |
| `maxPages` | `InfiniteQueryOptions.maxPages` |
| `fetchNextPage()` | `observer.fetchNextPage()` |
| `fetchPreviousPage()` | `observer.fetchPreviousPage()` |
| `hasNextPage` | `InfiniteSuccess.hasNextPage` |
| `hasPreviousPage` | `InfiniteSuccess.hasPreviousPage` |
| `isFetchingNextPage` | `InfiniteSuccess.isFetchingNextPage` |
| `isFetchingPreviousPage` | `InfiniteSuccess.isFetchingPreviousPage` |
| `data.pages` | `InfiniteData.pages` |
| `data.pageParams` | `InfiniteData.pageParams` |
| `data.pages.flat()` | `InfiniteData.flatten((page) => page.items)` |
| `queryClient.setQueryData` (infinite) | `client.setInfiniteQueryData<TData, TPageParam>(key, InfiniteData)` |
| `queryClient.getQueryData` (infinite) | `client.getInfiniteQueryData<TData, TPageParam>(key)` |

## DevTools

TanStack Query DevTools is a browser panel installed as a React component. Qora provides two surfaces:

- **In-app overlay**: install `qora_devtools_overlay` under `dev_dependencies`, wrap the app with `QoraInspector`. Absent from release builds. Available today.
- **IDE extension**: installs a dedicated tab in Flutter DevTools. Under active development, not yet published.

See [DevTools overview](/devtools/overview) for setup instructions.
