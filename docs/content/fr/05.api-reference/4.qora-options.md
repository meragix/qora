## Vue d'ensemble

`QoraOptions` configure le comportement d'une requête individuelle. Tout champ laissé à sa valeur par défaut hérite de `QoraClientConfig.defaultOptions`. Les champs sont fusionnés, les valeurs par requête ayant la priorité.

```dart [user_repository.dart]
await client.fetchQuery<User>(
  key: ['users', userId],
  fetcher: () => api.getUser(userId),
  options: QoraOptions(
    staleTime: Duration(minutes: 5),
    retryCount: 2,
  ),
);
```

---

## Constructeur

```dart
const QoraOptions({
  Duration staleTime = Duration.zero,
  Duration cacheTime = const Duration(minutes: 5),
  bool enabled = true,
  Object? dependsOn,
  int retryCount = 3,
  Duration retryDelay = const Duration(seconds: 1),
  Duration Function(int attemptIndex)? retryDelayCalculator,
  bool refetchOnWindowFocus = true,
  bool refetchOnReconnect = true,
  Duration? refetchInterval,
  bool? refetchOnMount,
  NetworkMode networkMode = NetworkMode.online,
  Object? Function(Object? data)? toJson,
})
```

---

## Champs

### `staleTime`

```dart
final Duration staleTime
```

**Défaut :** `Duration.zero`

Durée pendant laquelle les données sont considérées **fraîches** après un fetch réussi. Pendant la fenêtre de fraîcheur, `fetchQuery` et `watchQuery` ignorent complètement le réseau et retournent les données en cache.

Une fois `staleTime` écoulé, les données sont considérées **périmées** : Qora les retourne immédiatement mais déclenche aussi une revalidation en arrière-plan.

```dart [main.dart]
// Toujours périmé (défaut) : servir le cache, toujours revalider
staleTime: Duration.zero,

// Frais pendant 5 minutes : pas de réseau dans la fenêtre
staleTime: Duration(minutes: 5),

// Jamais périmé en pratique
staleTime: Duration(days: 365),
```

| Type de données              | Valeur recommandée         |
| ---------------------------- | -------------------------- |
| Prix en direct, feeds        | `Duration.zero`            |
| Profil utilisateur           | `Duration(minutes: 5)`     |
| Contenu article/post         | `Duration(minutes: 10)`    |
| Configuration applicative    | `Duration(hours: 1)`       |
| Données de référence statiques | `Duration(days: 1)`      |

---

### `cacheTime`

```dart
final Duration cacheTime
```

**Défaut :** `Duration(minutes: 5)`

Durée pendant laquelle une entrée **inactive** reste en cache après l'annulation de tous les abonnés `watchQuery`. Quand le timer GC se déclenche, l'entrée est supprimée ; le prochain accès repart de `Initial`.

```dart [main.dart]
// Garder en cache 10 minutes après avoir quitté l'écran
cacheTime: Duration(minutes: 10),

// Supprimer immédiatement à la désabonnement (ex. données de session sensibles)
cacheTime: Duration.zero,
```

:::tip
`staleTime` et `cacheTime` sont indépendants. Un pattern courant est `staleTime < cacheTime` : les données deviennent périmées (déclenchant une revalidation) mais restent en cache pour que la navigation retour soit instantanée.
:::

---

### `enabled`

```dart
final bool enabled
```

**Défaut :** `true`

Quand `false` :

- `fetchQuery` lève un `StateError` immédiatement
- `watchQuery` émet l'état actuel sans déclencher de fetch

Utile pour les **requêtes conditionnelles** qui ne doivent s'exécuter que lorsqu'un prérequis est satisfait :

```dart [main.dart]
// Fetch uniquement quand userId est non-null
options: QoraOptions(enabled: userId != null),
```

---

### `dependsOn`

```dart
final Object? dependsOn
```

**Défaut :** `null`

La clé d'une autre requête qui doit avoir des données `Success` avant que cette requête se déclenche. Fournit une dépendance réactive sans coordination manuelle :

| API           | Comportement quand la dépendance n'est pas résolue                            |
| ------------- | ----------------------------------------------------------------------------- |
| `watchQuery`  | S'abonne au stream de la dépendance ; se déclenche automatiquement à l'arrivée des données |
| `fetchQuery`  | Lève `StateError` immédiatement                                               |
| `prefetch`    | Ignore silencieusement                                                        |

```dart [main.dart]
// Étape 1 : récupérer la session (pas de dépendance)
client.watchQuery<Session>(
  key: ['session'],
  fetcher: () => api.getSession(),
);

// Étape 2 : récupérer l'utilisateur une fois que la session a des données
client.watchQuery<User>(
  key: ['user'],
  fetcher: () => api.getUser(session!.userId),
  options: QoraOptions(dependsOn: ['session']),
);
```

Voir le [guide des requêtes dépendantes](../04.recipes/07.dependent-queries.md) pour des exemples complets.

---

### `retryCount`

```dart
final int retryCount
```

**Défaut :** `3`

Nombre de tentatives de retry automatique après un échec de fetch. Mettre à `0` pour désactiver les retries.

```dart [main.dart]
retryCount: 0,  // échec immédiat à la première erreur
retryCount: 5,  // retry jusqu'à 5 fois
```

---

### `retryDelay`

```dart
final Duration retryDelay
```

**Défaut :** `Duration(seconds: 1)`

Délai de base entre les tentatives de retry. Par défaut Qora utilise un **backoff exponentiel** :

- Tentative 0 : `1 s`
- Tentative 1 : `2 s`
- Tentative 2 : `4 s`
- Tentative 3 : `8 s`

---

### `retryDelayCalculator`

```dart
final Duration Function(int attemptIndex)? retryDelayCalculator
```

**Défaut :** `null` (utilise le backoff exponentiel)

Calculateur de délai personnalisé. Reçoit l'index de tentative base-zéro et retourne le délai avant cette tentative.

```dart [main.dart]
// Délai fixe de 500 ms
retryDelayCalculator: (_) => const Duration(milliseconds: 500),

// Backoff exponentiel avec jitter
retryDelayCalculator: (i) => Duration(
  milliseconds: (500 * pow(2, i) + Random().nextInt(200)).toInt(),
),
```

---

### `refetchOnWindowFocus`

```dart
final bool refetchOnWindowFocus
```

**Défaut :** `true`

Indique si cette requête doit être rechargée quand l'application regagne le focus de fenêtre. Nécessite qu'un `LifecycleManager` soit enregistré sur le client.

---

### `refetchOnReconnect`

```dart
final bool refetchOnReconnect
```

**Défaut :** `true`

Indique si cette requête doit être rechargée quand la connexion réseau est rétablie. Nécessite qu'un `ConnectivityManager` soit enregistré sur le client.

---

### `refetchInterval`

```dart
final Duration? refetchInterval
```

**Défaut :** `null` (désactivé)

Quand défini, `watchQuery` recharge automatiquement à cette cadence tant qu'au moins un abonné est actif. Le timer s'arrête quand le dernier abonné annule.

```dart [dashboard_page.dart]
// Polling toutes les 30 secondes pendant que le widget est monté
options: QoraOptions(refetchInterval: Duration(seconds: 30)),
```

---

### `refetchOnMount`

```dart
final bool? refetchOnMount
```

**Défaut :** `null` (revient à `QoraClientConfig.refetchOnMount`)

Contrôle si `watchQuery` déclenche un fetch au premier abonnement :

| Valeur  | Comportement                                                        |
| ------- | ------------------------------------------------------------------- |
| `true`  | Toujours recharger au montage, même si les données sont fraîches    |
| `false` | Fetch uniquement si les données sont absentes ; ignorer si en cache |
| `null`  | Utiliser `QoraClientConfig.refetchOnMount` (défaut : `true`)        |

---

### `networkMode`

```dart
final NetworkMode networkMode
```

**Défaut :** `NetworkMode.online`

Contrôle le comportement de cette requête quand l'appareil n'a pas de connectivité :

| Valeur | Comportement |
| --- | --- |
| `NetworkMode.online` | Le fetch est **mis en pause** hors ligne (`FetchStatus.paused`). Rejoué automatiquement à la reconnexion. |
| `NetworkMode.always` | Le fetch **s'exécute toujours** quelle que soit la connectivité. Utiliser pour les fetchers locaux (base de données, mémoire). |
| `NetworkMode.offlineFirst` | Les données en cache sont retournées immédiatement. Une revalidation en arrière-plan est mise en file pour quand la connexion est rétablie. |

```dart [lib/data/user_repository.dart]
// Défaut : mise en pause hors ligne, rejeu à la reconnexion
options: const QoraOptions(networkMode: NetworkMode.online),

// Requête base de données locale : toujours exécuter, jamais mettre en pause
options: const QoraOptions(networkMode: NetworkMode.always),

// Servir le cache immédiatement, rafraîchir en arrière-plan quand en ligne
options: const QoraOptions(networkMode: NetworkMode.offlineFirst),
```

:::tip
Définir `NetworkMode.always` pour les requêtes sur SQLite, Hive ou tout store local : elles n'ont pas besoin de connectivité et ne doivent jamais être mises en pause.
:::

### `toJson`

```dart
final Object? Function(Object? data)? toJson
```

**Défaut :** `null`

Sérialiseur optionnel qui contrôle comment les données de la requête sont affichées dans l'overlay DevTools et l'extension IDE.

Quand omis, Qora applique l'ordre de résolution suivant automatiquement :

1. Les données qui sont déjà `Map`, `List`, `String`, `num`, `bool` ou `null`: passées telles quelles.
2. Tout objet avec une méthode `toJson()` (`json_serializable`, `freezed`, manuel): appelée dynamiquement.
3. Tout le reste: repli sur `.toString()`.

Fournir `toJson` explicitement uniquement quand la détection automatique est insuffisante, par exemple quand la méthode `toJson()` a une signature non standard ou retourne un type non encodable.

```dart [user_repository.dart]
// Sérialiseur explicite (remplace la détection automatique)
fetchQuery<User>(
  key: ['users', userId],
  fetcher: () => api.getUser(userId),
  options: QoraOptions(
    toJson: (data) => (data as User).toJson(),
  ),
);
```

## `getRetryDelay(int attemptIndex)`

```dart
Duration getRetryDelay(int attemptIndex)
```

Calcule le délai de retry pour l'index de tentative base-zéro donné. Délègue à `retryDelayCalculator` si défini, sinon applique le backoff exponentiel : `retryDelay × 2^attemptIndex`.

---

## `merge(QoraOptions? other)`

```dart
QoraOptions merge(QoraOptions? other)
```

Retourne un nouveau `QoraOptions` fusionnant `this` avec `other`. Les champs de `other` ont la priorité. Les champs nullable (`retryDelayCalculator`, `refetchInterval`, `refetchOnMount`) reviennent à `this` quand non définis dans `other`.

Utilisé en interne par `QoraClient` pour combiner les défauts globaux avec les options par requête.

---

## Recettes courantes

```dart [main.dart]
// Données en direct : toujours frais, polling toutes les 10 s
QoraOptions(
  staleTime: Duration.zero,
  refetchInterval: Duration(seconds: 10),
)

// Données de config : cache agressif
QoraOptions(
  staleTime: Duration(hours: 1),
  cacheTime: Duration(hours: 2),
  retryCount: 5,
)

// Requête conditionnelle
QoraOptions(enabled: currentUser != null)

// Désactiver les retries pour les mutations
QoraOptions(retryCount: 0)
```
