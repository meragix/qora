## Constructeur

```dart [main.dart]
QoraClient({QoraClientConfig? config})
```

Crée une nouvelle instance de client. Toutes les requêtes partagent le même cache et la même map de déduplication.

```dart [main.dart]
// Minimal (tous les défauts)
final client = QoraClient();

// Avec configuration complète
final client = QoraClient(
  config: QoraClientConfig(
    defaultOptions: QoraOptions(
      staleTime: Duration(minutes: 5),
      retryCount: 2,
    ),
    debugMode: kDebugMode,
    maxCacheSize: 200,
  ),
);
```

---

## Propriétés

### `config`

```dart
final QoraClientConfig config
```

La configuration résolue pour cette instance de client. Lecture seule.

### `cachedKeys`

```dart
Iterable<List<dynamic>> get cachedKeys
```

Toutes les clés de requête actuellement en cache (normalisées). Utile pour le débogage ou l'invalidation en masse.

```dart [main.dart]
for (final key in client.cachedKeys) {
  print(key); // [users, 1], [posts], ...
}
```

## Récupération

### `fetchQuery`

```dart
Future<T> fetchQuery<T>({
  required Object key,
  required Future<T> Function() fetcher,
  QoraOptions? options,
  CancelToken? cancelToken,
})
```

Récupère les données une seule fois et retourne le résultat.

**Comportement du cache :**

| État du cache      | Action                                                       |
| ------------------ | ------------------------------------------------------------ |
| Frais (non périmé) | Retourne les données en cache : pas d'appel réseau           |
| Périmé             | Retourne les données périmées immédiatement ; revalide en arrière-plan |
| Absent             | Fetch, attend et retourne le résultat                        |

Les appels concurrents avec la même clé sont **dédupliqués** : tous partagent une seule requête en cours. Les fetches échoués sont relancés avec backoff exponentiel jusqu'à `retryCount` fois.

```dart [user_repository.dart]
final user = await client.fetchQuery<User>(
  key: ['users', userId],
  fetcher: () => api.getUser(userId),
  options: QoraOptions(staleTime: Duration(minutes: 5)),
);
```

**Lève :**

- `StateError` si `QoraOptions.enabled` est `false`
- `StateError` si `QoraOptions.dependsOn` est défini et que la dépendance n'a pas encore de données
- `QoraCancelException` si `cancelToken` est annulé avant ou pendant le fetch
- L'exception du fetcher (après tous les retries) en cas d'échec

---

### `watchQuery`

```dart
Stream<QoraState<T>> watchQuery<T>({
  required Object key,
  required Future<T> Function() fetcher,
  QoraOptions? options,
  CancelToken? cancelToken,
})
```

Crée un `Stream<QoraState<T>>` réactif pour une requête.

**À l'abonnement :**

1. Émet immédiatement l'état en cache actuel
2. Déclenche un fetch si les données sont absentes ou périmées (selon `refetchOnMount`)
3. Configure un timer de polling si `refetchInterval` est défini
4. Pousse tous les futurs changements d'état : depuis n'importe quelle source

**À l'annulation (dernier abonné) :**

- Annule le timer de polling
- Planifie un timer GC pour l'entrée de cache

```dart [user_page.dart]
StreamBuilder<QoraState<User>>(
  stream: client.watchQuery<User>(
    key: ['users', userId],
    fetcher: () => api.getUser(userId),
    options: QoraOptions(refetchInterval: Duration(seconds: 30)),
  ),
  builder: (context, snapshot) {
    return switch (snapshot.data) {
      null || Initial()             => const SizedBox.shrink(),
      Loading(:final previousData)  => previousData != null
          ? UserCard(previousData)
          : const CircularProgressIndicator(),
      Success(:final data)          => UserCard(data),
      Failure(:final error)         => ErrorWidget('$error'),
    };
  },
)
```

### `prefetch`

```dart
Future<void> prefetch<T>({
  required Object key,
  required Future<T> Function() fetcher,
  QoraOptions? options,
  CancelToken? cancelToken,
})
```

Pré-chauffe le cache sans créer de stream. Sans effet si les données sont déjà fraîches.

```dart [user_list_page.dart]
onEnter: (_) => client.prefetch<User>(
  key: ['users', userId],
  fetcher: () => api.getUser(userId),
),
```

## Lecture du cache

### `getQueryData`

```dart
T? getQueryData<T>(Object key)
```

Retourne les meilleures données disponibles pour `key`, ou `null`. Retourne `Success.data`, ou `previousData` de `Loading`/`Failure`.

```dart [main.dart]
final user = client.getQueryData<User>(['users', userId]);
```

---

### `getQueryState`

```dart
QoraState<T> getQueryState<T>(Object key)
```

Retourne le `QoraState<T>` complet, ou `Initial<T>()` si non mis en cache.

```dart [main.dart]
final state = client.getQueryState<User>(['users', userId]);
if (state is Success<User>) {
  print('Ancienneté : ${state.age}');
}
```

## Écriture du cache

### `setQueryData`

```dart
void setQueryData<T>(Object key, T data)
```

Injecte des données directement dans le cache. Pousse instantanément `Success` vers tous les streams `watchQuery` actifs.

```dart [main.dart]
client.setQueryData(['users', userId], updatedUser);
```

---

### `restoreQueryData`

```dart
void restoreQueryData<T>(Object key, T? snapshot)
```

Effectue un rollback vers un snapshot précédent. Si `snapshot` est `null`, l'entrée est supprimée du cache.

```dart [main.dart]
final snapshot = client.getQueryData<User>(['users', userId]);
client.setQueryData(['users', userId], optimisticUser);

try {
  await api.updateUser(userId, payload);
} catch (_) {
  client.restoreQueryData(['users', userId], snapshot);
}
```

## Invalidation

### `invalidate`

```dart
void invalidate(Object key)
```

Marque une requête comme périmée. Les streams `watchQuery` actifs reçoivent immédiatement `Loading(previousData: ...)`. Le prochain accès déclenche un fetch frais.

```dart [main.dart]
await api.createPost(payload);
client.invalidate(['posts']);
```

---

### `invalidateWhere`

```dart
void invalidateWhere(bool Function(List<dynamic> key) predicate)
```

Invalide toutes les requêtes dont la clé satisfait `predicate`.

```dart [main.dart]
// Toutes les requêtes utilisateur
client.invalidateWhere((key) => key.firstOrNull == 'users');

// Utilisateur spécifique + ses sous-ressources
client.invalidateWhere(
  (key) => key.length >= 2 && key[0] == 'users' && key[1] == userId,
);
```

---

### `invalidateQueries`

```dart
void invalidateQueries({required QueryFilter filter})
```

Invalidation en masse avec un `QueryFilter` : un prédicat plus riche qu'`invalidateWhere` qui reçoit également l'état actuel et les options du dernier fetch réussi.

```dart
typedef QueryFilter = bool Function(
  String key,
  QoraState<dynamic> state,
  QoraOptions? lastOptions,
);
```

```dart [main.dart]
// Invalider toutes les requêtes en mode online pas encore en Success
client.invalidateQueries(
  filter: (key, state, opts) =>
      opts?.networkMode == NetworkMode.online && state is! Success,
);

// Invalider toutes les requêtes utilisateur dont les données ont plus de 10 minutes
client.invalidateQueries(
  filter: (key, state, opts) {
    if (state is! Success) return false;
    return key.contains('users') &&
        DateTime.now().difference(state.updatedAt) > const Duration(minutes: 10);
  },
);
```

## Suppression du cache

### `removeQuery`

```dart
void removeQuery(Object key)
```

Supprime une entrée du cache et annule toute requête en attente. Le prochain accès repart de `Initial`.

```dart [main.dart]
await api.deletePost(postId);
client.removeQuery(['posts', postId]);
```

---

### `clear`

```dart
void clear()
```

Vide l'intégralité du cache et annule toutes les requêtes en attente.

```dart [main.dart]
// À la déconnexion
client.clear();
```

## Hydratation

### `queueHydration`

```dart
void queueHydration(Object key, dynamic data, {DateTime? updatedAt})
```

Met en file une valeur pré-désérialisée pour injection différée dans le cache typé. La valeur est injectée au premier appel API typé (`fetchQuery<T>`, `watchQuery<T>`, etc.) pour cette clé, reportant le cast Dart jusqu'à ce que le type `T` soit connu.

C'est le mécanisme partagé utilisé par `PersistQoraClient.hydrate()` et `SsrHydrator.hydrate()`. L'utilisation directe est réservée aux scénarios avancés (ex. sources d'hydratation personnalisées).

```dart [main.dart]
// Pré-peupler manuellement le cache avant runApp
client.queueHydration(['users', 1], User(id: 1, name: 'Ada'));

// Avec un timestamp personnalisé pour que staleTime s'évalue correctement
client.queueHydration(
  ['config'],
  appConfig,
  updatedAt: DateTime.now().subtract(const Duration(minutes: 2)),
);
```

---

### `hydrateQuery`

```dart
void hydrateQuery<T>(Object key, T data, {DateTime? updatedAt})
```

Injecte un `Success<T>` typé directement dans une entrée `Initial` du cache. Sans effet si l'entrée a déjà des données. Utilisé en interne ; préférer `queueHydration` quand le type n'est pas connu à l'appel.

Map<String, dynamic> debugInfo()

## Cycle de vie

### `attachLifecycleManager`

```dart
void attachLifecycleManager(LifecycleManager manager)
```

Enregistre un `LifecycleManager` auprès du client. Quand l'application passe à `LifecycleState.resumed`, le client revalide toutes les entrées périmées ayant des abonnés actifs. `QoraScope` appelle cela automatiquement quand un `FlutterLifecycleManager` est fourni. L'utilisation directe est uniquement nécessaire dans des scénarios d'intégration personnalisés.

---

### `dispose`

```dart
void dispose()
```

Libère toutes les ressources. Arrête le timer d'éviction, dispose toutes les entrées du cache, annule les requêtes en attente. Le client **ne doit pas** être utilisé après cet appel.

```dart [main.dart]
// Dans un teardown DI ou un tearDown de test
client.dispose();
```

## Configuration recommandée

```dart [lib/core/qora.dart]
import 'package:flutter/foundation.dart';
import 'package:qora/qora.dart';

/// Singleton : créer une fois, injecter partout via DI ou QoraScope.
final qoraClient = QoraClient(
  config: QoraClientConfig(
    defaultOptions: QoraOptions(
      staleTime: Duration(minutes: 5),
      cacheTime: Duration(minutes: 10),
      retryCount: 3,
    ),
    debugMode: kDebugMode,
    maxCacheSize: 200,
    refetchOnMount: true,
    errorMapper: (error, stack) => QoraException(
      '$error',
      originalError: error,
      stackTrace: stack,
    ),
  ),
);
```
