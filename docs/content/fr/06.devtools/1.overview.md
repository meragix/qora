Utilisez l'[Overlay intégré](/devtools/overlay) pour une expérience de débogage stable dès aujourd'hui. L'extension sera annoncée quand elle atteindra une version stable.

Qora fournit deux surfaces d'outillage développeur. Les deux consomment le même protocole d'événements. Aucune ne nécessite de modifications de configuration dans les builds de production.

::card-group
  ::card
  ---
  title: Overlay intégré
  icon: i-heroicons-squares-2x2
  to: /devtools/overlay
  ---
  Un panneau déplaçable rendu à l'intérieur de l'app en cours d'exécution. Cette surface est absente des builds de release.
  ::

  ::card
  ---
  title: Extension IDE
  icon: i-heroicons-command-line
  to: /devtools/ide-extension
  ---
  Un onglet natif dans Flutter DevTools pour VS Code et IntelliJ. Inspecte le cache complet et envoie des commandes à l'appareil. **En développement — pas encore stable.**
  ::
::

## Comparaison

|                                | Overlay intégré                        | Extension IDE                                        |
| ------------------------------ | -------------------------------------- | ---------------------------------------------------- |
| **Configuration**              | Envelopper l'app avec `QoraInspector`  | Appeler `QoraDevtools.setup(client)`                 |
| **Où s'exécute-t-il**          | Dans le processus de l'app             | Dans Flutter DevTools (VS Code / IntelliJ)           |
| **Transport**                  | Direct, même processus                 | Dart VM Service (WebSocket)                          |
| **Timeline des mutations**     | Oui                                    | Oui                                                  |
| **Inspecteur de cache**        | Oui                                    | Oui (live + snapshot)                                |
| **Mises à jour optimistes**    | Oui                                    | Oui                                                  |
| **Data Diff**                  | Oui                                    | Oui                                                  |
| **Refetch / Invalidate**       | Oui                                    | Oui                                                  |
| **Moniteur d'activité réseau** | Non disponible                         | Oui                                                  |
| **Métriques de performance**   | Non disponible                         | Oui                                                  |
| **Graphe de dépendances**      | Non disponible                         | Oui                                                  |
| **Arbre de widgets**           | Non disponible                         | Prévu                                                |
| **Impact en production**       | Nul (garde `kDebugMode`)               | Nul (supprimé par le tree shaker)                    |
| **Fonctionne sur appareil physique** | Oui                              | Oui (via USB)                                        |

## Fonctionnement

Les deux surfaces consomment le flux d'événements produit par `QoraTracker`, l'interface d'observabilité de `QoraClient`. En production, aucun tracker n'est configuré. `QoraClient` utilise par défaut `NoOpTracker`, qui a un overhead d'exécution nul.

```text
QoraClient
    │
    ├── NoOpTracker    (production)       silencieux, aucun événement émis
    ├── VmTracker      (extension IDE)    émet des événements via Dart VM Service
    └── OverlayTracker (overlay)          émet des événements en mémoire
```

Pour l'extension IDE, configurer le tracker via `QoraDevtools.setup`. Pour l'overlay seul, passer le tracker directement au constructeur de `QoraClient`. En production, passer `null` ou omettre le paramètre `tracker` pour activer `NoOpTracker`.
