## Installation

Ajouter `qora_devtools_overlay` dans `dev_dependencies`. Cela exclut le package des bundles de production.

```yaml [pubspec.yaml]
dev_dependencies:
  qora_devtools_overlay: ^1.0.0
```

---

## Configuration

Créer un `OverlayTracker` et le passer à la fois à `QoraClient` et à `QoraInspector`. Passer la référence du client à `QoraInspector` pour activer les boutons d'action de l'inspecteur.

```dart [main.dart]
import 'package:flutter/foundation.dart';
import 'package:qora/qora.dart';
import 'package:qora_devtools_overlay/qora_devtools_overlay.dart';

void main() {
  final tracker = OverlayTracker();

  final client = QoraClient(
    // null en mode release : QoraClient utilise NoOpTracker, aucun événement émis.
    tracker: kDebugMode ? tracker : null,
  );

  runApp(
    QoraInspector(
      tracker: tracker,
      // Requis pour que les boutons d'action (Refetch, Invalidate, Remove, etc.)
      // opèrent sur le cache en direct. Omettre cela désactive la section ACTIONS.
      client: kDebugMode ? client : null,
      child: MyApp(client: client),
    ),
  );
}
```

En mode debug, un bouton **Q** est rendu dans le coin inférieur droit de l'écran. Appuyer dessus ouvre le panneau.

::callout{type="warning"}
Le paramètre `client` est optionnel. `QoraInspector` opère en tant qu'observateur en lecture seule quand `client` n'est pas fourni. La section **ACTIONS** n'est pas rendue dans ce cas.
::

::callout{type="info"}
`QoraInspector` ne rend rien dans les builds de release. La garde est appliquée au niveau du widget : le sous-arbre overlay n'est pas construit quand `kDebugMode` est false.
::

### Avec l'extension IDE

Quand l'overlay et l'extension IDE sont nécessaires simultanément, utiliser `QoraDevtools.setup` avec `additionalTrackers`. Les deux trackers reçoivent des événements identiques via un `MultiTracker` partagé.

```dart [main.dart]
import 'package:flutter/foundation.dart';
import 'package:qora_devtools_extension/qora_devtools_extension.dart';
import 'package:qora_devtools_overlay/qora_devtools_overlay.dart';

void main() {
  final overlay = OverlayTracker();
  final client  = QoraClient();

  if (kDebugMode) {
    QoraDevtools.setup(client, additionalTrackers: [overlay]);
  }

  runApp(
    QoraInspector(
      tracker: overlay,
      client: kDebugMode ? client : null,
      child: MyApp(client: client),
    ),
  );
}
```

## Comportement responsive

La disposition de l'overlay s'adapte à la largeur d'écran disponible.

- **600 px et plus** (tablette, desktop) : disposition trois colonnes avec tous les panneaux visibles simultanément.
- **Moins de 600 px** (téléphone) : navigation mono-colonne avec navigation vers les vues inspecteur et timeline.

## Sécurité en production

Trois mécanismes indépendants garantissent l'absence de l'overlay en production :

### 1. `dev_dependencies`

Le package n'est pas inclus dans le bundle de release car il est déclaré sous `dev_dependencies`.

### 2. Garde du widget

```dart
@override
Widget build(BuildContext context) {
  if (!kDebugMode) return child; // rien n'est construit en release
  return _buildOverlay();
}
```

### 3. NoOpTracker en release

`QoraClient` est construit avec `tracker: kDebugMode ? tracker : null`. En mode release, `null` active `NoOpTracker`. Ses méthodes sont des no-ops et aucun événement de suivi n'est émis.

Résultat : aucun impact d'exécution depuis le chemin du tracker overlay en production.
