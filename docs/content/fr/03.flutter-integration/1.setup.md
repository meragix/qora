## Installation

Ajouter `qora_flutter` à `pubspec.yaml` :

```yaml [pubspec.yaml]
dependencies:
  qora_flutter: ^1.0.0
```

Puis récupérer les dépendances :

```bash
flutter pub get
```

:::tip
`qora_flutter` inclut automatiquement `qora` et `connectivity_plus` comme dépendances. Ces packages n'ont pas besoin d'être ajoutés séparément.
:::

---

## Créer le client

`QoraClient` est le cache central et l'orchestrateur de requêtes. Créer **une seule instance par application** (ou par scope DI) :

```dart [main.dart]
import 'package:qora/qora.dart';

final client = QoraClient(
  config: QoraClientConfig(
    defaultOptions: QoraOptions(
      staleTime: Duration(minutes: 5),
      cacheTime: Duration(minutes: 10),
      retryCount: 3,
    ),
    debugMode: kDebugMode,
  ),
);
```

:::tip
Définir `debugMode: kDebugMode` pour obtenir des logs détaillés en développement sans coût de performance en production.
:::

## Encapsuler l'application avec QoraScope

`QoraScope` est un `StatefulWidget` qui expose le client à tous les descendants via un `InheritedWidget` interne. Le placer aussi près de la racine que possible :

```dart [main.dart]
import 'package:flutter/material.dart';
import 'package:qora_flutter/qora_flutter.dart';

void main() {
  final client = QoraClient(
    config: const QoraClientConfig(debugMode: kDebugMode),
  );

  runApp(
    QoraScope(
      client: client,
      child: const MyApp(),
    ),
  );
}
```

Tout widget descendant peut désormais accéder au client via `context.qora` ou `QoraScope.of(context)`.

## Optionnel : gestionnaires de cycle de vie et de connectivité

Pour le refetch automatique à la reprise de l'application et à la reconnexion réseau, passer les gestionnaires intégrés à `QoraScope` :

```dart [main.dart]
void main() {
  final client = QoraClient();

  runApp(
    QoraScope(
      client: client,
      lifecycleManager: FlutterLifecycleManager(
        qoraClient: client,
        // Invalider toutes les requêtes si l'app était en arrière-plan depuis plus de 30 s
        refetchInterval: const Duration(seconds: 30),
      ),
      connectivityManager: FlutterConnectivityManager(),
      child: const MyApp(),
    ),
  );
}
```

| Gestionnaire                 | Comportement                                                                                              |
| ---------------------------- | --------------------------------------------------------------------------------------------------------- |
| `FlutterLifecycleManager`    | Invalide toutes les requêtes quand l'application reprend après une pause supérieure à `refetchInterval`.  |
| `FlutterConnectivityManager` | Émet des événements `NetworkStatus` ; la logique de reconnexion est déléguée à `QoraClient` via `reconnectStrategy`. |

`QoraScope` appelle automatiquement `start()` et `dispose()` sur les deux gestionnaires : aucun câblage manuel du cycle de vie n'est requis.
