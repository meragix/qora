`QoraScope` est le point de vérité unique pour le `QoraClient` dans Flutter. Encapsuler l'application (ou un sous-arbre) avec ce widget une seule fois, et chaque descendant accède au même cache et au même pipeline de requêtes.

---

## Anatomie

```dart [main.dart]
QoraScope(
  // Requis : le cache partagé et le moteur de requêtes
  client: client,

  // Optionnel : invalide les requêtes à la reprise de l'application
  lifecycleManager: FlutterLifecycleManager(qoraClient: client),

  // Optionnel : fournisseur de signal pur pour la connectivité réseau
  connectivityManager: FlutterConnectivityManager(),

  child: const MyApp(),
)
```

`QoraScope` est un `StatefulWidget`. Il appelle `start()` sur chaque gestionnaire dans `initState` et appelle `dispose()` sur les deux gestionnaires **et** sur le client lui-même dans `dispose`. Le cycle de vie n'est jamais géré manuellement.

---

## Accéder au client

### `context.qora` (recommandé)

Le chemin le plus court vers le client depuis n'importe quel widget :

```dart [user_list_screen.dart]
// Invalider une requête spécifique après une mutation
context.qora.invalidate(['posts', postId]);

// Invalider par prédicat
context.qora.invalidateWhere((key) => key.firstOrNull == 'posts');

// Mise à jour optimiste
final snapshot = context.qora.getQueryData<User>(['users', userId]);
context.qora.setQueryData(['users', userId], updatedUser);
```

### `QoraScope.of(context)`

Équivalent à `context.qora`. Lève une `FlutterError` descriptive si aucun ancêtre `QoraScope` n'est trouvé dans l'arbre :

```dart [user_repository.dart]
final client = QoraScope.of(context);
await client.fetchQuery<List<Post>>(
  key: ['posts'],
  fetcher: api.getPosts,
);
```

### `QoraScope.maybeOf(context)`

Retourne `null` au lieu de lever une exception quand `QoraScope` est absent. Utile quand le scope est optionnel dans une partie de l'arbre :

```dart [optional_widget.dart]
final client = QoraScope.maybeOf(context);
if (client == null) return const SizedBox.shrink();
```

### `context.qoraOrNull`

Raccourci pour `QoraScope.maybeOf(context)` :

```dart [nav_bar.dart]
context.qoraOrNull?.invalidate(['notifications']);
```

## Scopes imbriqués

Plusieurs widgets `QoraScope` peuvent être imbriqués pour isoler les caches dans différentes parties de l'application : par exemple, des caches séparés pour les routes publiques et authentifiées :

```dart [app.dart]
QoraScope(
  client: publicClient,
  child: Navigator(
    pages: [
      MaterialPage(child: const HomeScreen()),
      MaterialPage(
        child: QoraScope(
          // Cache séparé pour les routes authentifiées
          client: authClient,
          child: const DashboardScreen(),
        ),
      ),
    ],
  ),
)
```

Chaque `QoraScope` gère son propre client : les requêtes sont entièrement indépendantes entre les scopes.

## Gestionnaires de cycle de vie et de connectivité

### `FlutterLifecycleManager`

Observe les changements d'`AppLifecycleState` via `WidgetsBindingObserver`. Quand l'application reprend après une pause supérieure à `refetchInterval`, il appelle `client.invalidateWhere((_) => true)` :

```dart [main.dart]
FlutterLifecycleManager(
  qoraClient: client,
  refetchInterval: const Duration(minutes: 1), // défaut : 5 s
)
```

Les widgets `QoraBuilder` actifs reçoivent l'état `Loading(previousData: data)` résultant et déclenchent immédiatement des requêtes de refetch. Les utilisateurs voient les données périmées instantanément pendant que les données fraîches se chargent en arrière-plan.

### `FlutterConnectivityManager`

Un **fournisseur de signal pur** : il convertit les événements `connectivity_plus` en valeurs `NetworkStatus` uniquement. `QoraScope` le câble automatiquement au client via `attachConnectivityManager()` après l'appel de `start()`.

Toute la logique de reconnexion (relecture des requêtes, relecture de la file de mutations hors-ligne, traitement par lots du thundering-herd) s'exécute dans `QoraClient`, et non dans le gestionnaire :

```dart [main.dart]
// Pas de référence QoraClient requise : fournisseur de signal pur
FlutterConnectivityManager()
```

Nécessite `connectivity_plus` dans `pubspec.yaml` :

```yaml [pubspec.yaml]
dependencies:
  connectivity_plus: ^6.0.0
```

:::tip
`FlutterConnectivityManager` n'invalide plus les requêtes directement. Le comportement de reconnexion est contrôlé par `QoraClientConfig.reconnectStrategy` et `QoraOptions.networkMode` par requête. Implémenter un `ConnectivityManager` personnalisé pour sourcer les événements de connectivité depuis un package différent.
:::

### Accéder au `ConnectivityManager` depuis les descendants

`QoraScope.connectivityManagerOf(context)` expose le gestionnaire aux widgets plus profonds dans l'arbre (utilisé en interne par `NetworkStatusBuilder`) :

```dart [lib/ui/offline_badge.dart]
final manager = QoraScope.connectivityManagerOf(context);
final status = manager?.currentStatus ?? NetworkStatus.unknown;
```

## Référence API

| Propriété / Méthode                        | Type                    | Description                                                          |
| ------------------------------------------ | ----------------------- | -------------------------------------------------------------------- |
| `client`                                   | `QoraClient`            | **Requis.** Le client partagé pour ce scope.                         |
| `lifecycleManager`                         | `LifecycleManager?`     | Optionnel. Refetch à la reprise de l'application.                    |
| `connectivityManager`                      | `ConnectivityManager?`  | Optionnel. Fournisseur de signal pur : pas de référence `QoraClient` requise. |
| `child`                                    | `Widget`                | **Requis.** Le sous-arbre de widgets.                                |
| `QoraScope.of(context)`                    | `QoraClient`            | Retourne le client ; lève une exception si absent.                   |
| `QoraScope.maybeOf(context)`               | `QoraClient?`           | Retourne le client, ou `null` si absent.                             |
| `QoraScope.connectivityManagerOf(context)` | `ConnectivityManager?`  | Retourne le gestionnaire actif ; utilisé par `NetworkStatusBuilder`. |
| `context.qora`                             | `QoraClient`            | Raccourci pour `QoraScope.of(context)`.                              |
| `context.qoraOrNull`                       | `QoraClient?`           | Raccourci pour `QoraScope.maybeOf(context)`.                         |
